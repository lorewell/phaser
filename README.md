# 🛠️ 《砖块幸存者》开发详细设计文档

## 1. 核心概述

* **游戏类型**：Roguelike + 动作打砖块
* **开发引擎**：Phaser 3 (JavaScript/TypeScript)
* **核心循环**：
1. **战斗阶段**：玩家控制挡板反弹球，击碎带有攻击性或特殊属性的砖块。
2. **清波判定**：当本关砖块清空，游戏暂停。
3. **三选一升级**：玩家从随机生成的三个 Buff 中选择一个强化自身。
4. **难度阶梯**：进入下一波，砖块血量、速度和攻击频率提升。
5. **失败/结算**：生命值归零，永久死亡，结算奖励用于局外成长。



---

## 2. 第一阶段：实现“打砖块”核心 (MVP)

这是游戏的地基。首先要让球动起来，挡板能接球。

### 2.1 环境配置

* 使用 `Phaser.Physics.Arcade` 物理引擎。
* 画布尺寸建议：`450x800` (竖屏，适合移动端/网页)。

### 2.2 关键对象构建

* **挡板 (Paddle)**：
* 设置为 `immovable: true`。
* 绑定输入：`this.input.on('pointermove', ...)` 随鼠标/手指移动。


* **球 (Ball)**：
* 设置 `collideWorldBounds: true`。
* 设置 `bounce: 1`（完美弹性）。


* **砖块组 (Bricks Group)**：
* 使用 `StaticGroup` 提升性能。



### 2.3 核心碰撞逻辑

```javascript
// 处理球与砖块碰撞
this.physics.add.collider(this.ball, this.bricks, (ball, brick) => {
    brick.takeDamage(this.playerPower); // 允许砖块有多点血量
    if (brick.isDead) {
        brick.destroy();
        this.checkWaveClear(); // 每次击碎检查是否进入下一关
    }
});

```

## 3. 第二阶段：Roguelike 循环系统

这是游戏从“普通打砖块”进化到“肉鸽”的关键。

### 3.1 状态机管理

定义游戏状态：`PLAYING` (战斗), `UPGRADING` (升级中), `GAMEOVER` (失败)。

### 3.2 升级阶段逻辑 (Three-Choice UI)

当检测到砖块清空：

1. `this.physics.pause()` 暂停画面。
2. **生成随机 Buff 池**：从预设的 `BuffList` 中随机抽取 3 个。
3. **UI 渲染**：
* 弹出半透明遮罩层。
* 生成三个交互式卡片（Container 对象）。
* 点击卡片后，修改 `PlayerConfig` 变量，清除 UI，执行 `this.physics.resume()`。

## 4. 第三阶段：数值构筑与 Buff 系统

你需要设计一个可扩展的 Buff 库。

### 4.1 基础属性类 Buff

| 属性名 | 效果 | 实现逻辑 |
| --- | --- | --- |
| **磁力挡板** | 挡板变长 | `paddle.displayWidth += 20` |
| **超频球速** | 球的基础速度提升 | `ballSpeed *= 1.1` |
| **合金外壳** | 增加挡板生命值上限 | `maxHealth += 1` |

### 4.2 特殊技能类 Buff (重点)

* **分裂球 (Multiball)**：当球撞击挡板时，有 20% 概率生成一个新的球。
* **燃烧轨迹 (Fire Trail)**：球经过的地方产生伤害粒子。
* **激光发射 (Auto-Laser)**：挡板每 3 秒自动向上发射一道激光，伤害它碰到的第一个砖块。

---

## 5. 第四阶段：敌人（砖块）的多样性

砖块不再是死物，而是会反击的“怪物”。

### 5.1 砖块类型设计

| 属性名 | 效果 |
| --- | --- | --- |
| **普通砖块** | 1 次撞击即碎，灰色 |
| **装甲砖块** | 需要撞击 3 次，颜色随生命值变淡。银色填充白色边框，被撞击一次后没有白色边框，俩次后变为灰色，三次即碎 |
| **自爆砖块** | 碎裂时向正下方发射 1 枚子弹，玩家需控制挡板躲避，命中则减少玩家生命值，红色 |
| **相位幽灵** | 每隔 3 秒在“实体”与“虚幻”间切换，虚幻状态时球会直接穿过去，实体时虚线框灰色填充，虚幻时只有虚线框 |


### 5.2 难度缩放 (Scaling)

每过一关，通过全局变量 `waveCount` 自动调整：

* `brickHealth = 1 + Math.floor(waveCount / 5)`
* `enemyBulletSpeed = 150 + (waveCount * 5)`

## 6. 第五阶段：游戏表现力提升 (Juice)

为了让游戏“爽”，必须加入视觉反馈。

* **屏幕抖动 (Screen Shake)**：当球撞击砖块或挡板受损时，调用 `this.cameras.main.shake(100, 0.01)`。
* **粒子效果 (Particles)**：
* 砖块破碎时迸发出对应颜色的碎片。
* 球运动时的拖尾烟雾。


* **顿帧 (Hit Stop)**：强力撞击时，游戏逻辑暂停 0.05 秒，增加打击感。

## 7. 推荐开发计划表 (Step-by-Step)

* [ ] **第一天：基础搭建**。实现挡板移动、球的反弹、一排简单的砖块销毁。
* [ ] **第二天：数值框架**。加入生命值系统、得分系统，实现“清完一波刷新一波”的逻辑。
* [ ] **第三天：升级系统**。制作三选一 UI 界面，实现“球变大”和“板变长”两个基础 Buff。
* [ ] **第四天：扩展 Buff 库**。实现至少 5 种不同的随机升级，并测试它们叠加后的平衡性。
* [ ] **第五天：怪物化砖块**。给砖块加入生命值条和简单的开火逻辑。
* [ ] **第六天：打磨与特效**。加入粒子、音效、屏幕震动和转场动画。
