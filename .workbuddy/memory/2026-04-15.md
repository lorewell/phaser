## 代码模块化重构
将单文件 game.js（~1130行）重构为模块化结构：

### 新文件结构
```
src/
├── constants.js              # 游戏配置、常量、枚举
├── entities/
│   └── Fighter.js           # 角色实体类
├── managers/
│   ├── InputManager.js      # 输入管理（键盘、攻击解析）
│   ├── UIManager.js         # UI 管理（血条、怒气、计时器）
│   ├── EffectsManager.js    # 特效管理（火花、伤害数字、技能名）
│   └── RoundManager.js      # 回合管理（计时、KO、胜负判定）
├── scenes/
│   └── FightScene.js        # 主场景（精简版，调用各管理器）
└── main.js                  # 游戏入口
```

### 主要改进
- **关注点分离**：输入、UI、特效、回合逻辑各自独立
- **可维护性**：每个模块职责清晰，便于后续扩展
- **常量集中**：GAME_CONFIG、ATTACK_TYPES、PLAYER_IDS 等统一管理
- **删除旧文件**：game.js 已移除，index.html 更新为模块化引入

### 兼容性
- 保留原有游戏逻辑和玩法
- SVG 精灵图系统完全保留
- 所有动画、特效、判定逻辑保持一致
